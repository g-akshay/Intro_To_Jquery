What is $

$ is a alias for jQuery

  • All of jQuery lives inside a single global function which is 'jQuery'
  • Other libraries using $ are prototype, mootools, etc.
  • Use jQuery.noConflict() to avoid conflicts and to handover '$' to other libraries.
  • One can use anonymous functions as well.
(function($){
...
})(jQuery);

Selectors

$() is query selector which returns a collection of matched elements found in the DOM based on passed argument(s)

  • id, class
  • HTML tag
  • Attribute
  • Pseudo-Selectors
  • Other selectors
$('.class, #id, name[elem_name]');
$('input, select, radio, h1');
$('[name="elem_name"],link[rel="stylesheet"]');
$('tr:odd, div:visible');

Optimizing Selectors

  • Begin your selector with an ID if possible 
    // Fast:
$( "#container div.myElem" );
 
// Super-fast:
$( "#container" ).find( "div.myElem" );
  • Be specific on the right-hand side of your selector, and less specific on the left 
    // Unoptimized:
$( "div.data .myElem" );
 
// Optimized:
$( ".data td.myElem" );
  • Avoid excessive specificity 
    $(".box table#attendees tbody tr td.myElem");
 
// Better: Drop the middle if possible.
$( "#attendees td.myElem" );
  • Avoid Selector Repetition 
    $("p").css("color", "blue");
$("p").css("font-size", "12px");
$("p").text("Change my text.");

// instead take advantage of jQuery's chaining
$("p").css({ "color"     : "blue",
             "font-size" : "12px"})
      .text("Change my text.");

When to use What

' == ' vs ' === '

  • === and !== are strict equality operator: considers value equal when they are of same type
  • == and != are lenient equator (EVIL TWINS): tries to convert values of different types, before comparing like strict equality
'10' === 10  // false
'10' == 10  // true
// some WIERED behaviour of EVIL TWINS
"" == '0'   /*false*/  false == 'false' /*false*/
0  == ""   /*true*/    false == null    /*false*/
0  == '0'  /*true*/    false == '0'     /*true*/
false == undefined  /*false*/
null  == undefined  /*true*/

.attr() vs .data()

data set with .attr()

  • Accessible from .attr('data-some-name')
  • Attribute and value are visible on the element upon inspection
  • data- attributes signify server-side data
  • Objects can not be stored

Accessing with .data()

  • Skip the data- part .data('some-name')
  • Understands camel-casing convention .data('someName')

.attr() vs .data()

data set with .data()

  • Accessible only from .data(name)
  • Not accessible from .attr() or anywhere else
  • Not visible on the element when inspected
  • Most important => Objects can be stored
  • Data can not be accessed without using jQuery
  • Thumb Rule: use .attr() to extract metadata that's been added to the markup and use .data() to get/set dynamic application specific values.
